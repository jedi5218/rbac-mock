# Глосарій

## Організації

**Ієрархія організацій** — організації утворюють дерево через батьківсько-дочірні відносини. Кожна організація має щонайбільше одного батька. Коренева організація батька не має.

**Піддерево організації** — організація та всі її нащадки (дочірні, онуки тощо). Використовується для адмін-скоупінгу: адмін організації керує всім своїм піддеревом.

**Вертикальна лінія** — для даної організації X: множина предків X (вгору до кореня) плюс нащадки X. Виключає сусідів (інших дітей батька X). Використовується для визначення меж [зовнішнього поширення](/wiki/propagation).

## Ролі

**DAG ролей** — ролі посилаються на інші ролі через включення, утворюючи направлений ациклічний граф. Роль успадковує всі дозволи від включених ролей транзитивно. Цикли виявляються та відхиляються перед вставкою.

**Включення ролі** — направлене ребро в DAG ролей. Якщо роль A включає роль B, то A успадковує всі дозволи B. Включення можуть перетинати межі організацій, якщо це дозволено обміном.

**Транзитивне замикання** — повна множина ролей, досяжних з даної ролі через всі ланцюжки включень. Використовується для обчислення ефективних дозволів.

**Роль @members** — автоматично створена системна роль, одна на організацію (`is_org_role = true`). Кожен користувач організації автоматично до неї призначений. Кожна роль @members включає роль @members батьківської організації, утворюючи ланцюжок для поширення базових дозволів по дереву. Не може бути вручну змінена або видалена.

## Дозволи

**Бітова маска дозволів** — ціле число, що кодує множину дозволів для конкретного типу ресурсу. Кожна бітова позиція представляє окремий дозвіл. Поточні типи ресурсів:

| Тип | Біт 1 | Біт 2 | Біт 4 |
|---|---|---|---|
| document | read | write | — |
| video | view | comment | stream |

**Агрегація BIT_OR** — метод об'єднання дозволів з кількох ролей. Якщо роль A надає `read` (біт 1), а роль B — `write` (біт 2), ефективний дозвіл — `read + write` (біти 1 | 2 = 3). Дозволи лише накопичуються — одна роль не може відкликати те, що надала інша.

**Ефективні дозволи** — фінальна множина дозволів користувача на кожен ресурс, обчислена обходом DAG ролей від усіх безпосередньо призначених ролей, збором усіх пар роль-ресурс та агрегацією через BIT_OR.

## Обміни

**Обмін організацій** — двостороння угода між двома організаціями для міжорганізаційного обміну ролями. Кожна сторона може надавати конкретні ролі партнеру. Зберігається з канонічним порядком (`org_a_id < org_b_id`) для запобігання дублікатів.

**Надана роль** — роль, яку організація зробила доступною для свого партнера по обміну. Лише організація-власник може надавати або відкликати її. Після надання адміни партнера можуть включати її у свої ролі.

**Зовнішня роль** — роль, що належить організації поза ланцюжком предків/нащадків поточного користувача. Зовнішні ролі можна включати лише якщо вони надані через активний обмін.

**Зовнішнє перетинання** — подія під час обходу DAG ролей, коли включення ролі переходить до організації поза вертикальною лінією поточної організації. Після перетинання лише вертикальна лінія організації перетинання залишається досяжною. Див. [Зовнішнє поширення](/wiki/propagation).

## Концепції архітектури

Ці терміни з'являються в огляді [продакшен-архітектури](/wiki/architecture).

**Матеріалізований шлях** — рядок, що кодує позицію організації в ієрархії (наприклад, `/root/div-a/team-alpha/`). Замінює рекурсивні запити на індексовані рядкові операції для пошуку піддерев та предків.

**Таблиця замикання** — таблиця, що зберігає всі пари (предок, нащадок, глибина) для ієрархії. Альтернатива матеріалізованим шляхам. Попередньо обчислює транзитивні відносини для O(1) пошуку.

**Матеріалізовані ефективні дозволи** — попередньо обчислена таблиця, що відображає (user_id, resource_id) на біти дозволів. Оновлюється асинхронно при зміні графа ролей. Робить перевірки доступу O(1) замість потреби обходу DAG на запит.

**CQRS (Command Query Responsibility Segregation)** — архітектурний патерн розділення операцій читання та запису. В цій системі: читання (перевірки доступу) звертається до попередньо обчисленої таблиці; запис (адмін-операції) оновлює таблиці-джерела та запускає фонове переобчислення.

**Область інвалідації кешу** — множина кешованих записів, що потребують оновлення після конкретної зміни. Варіюється від одного користувача (призначення ролі) до всіх користувачів у піддеревах обох організацій (створення/закриття обміну).
